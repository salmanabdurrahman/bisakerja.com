## Kontrak Field Kritis yang Wajib Dipakai Frontend

- **Auth login/refresh**: `access_token`, `refresh_token`, `expires_in`, `token_type`.
- **Auth me**: `id`, `email`, `name`, `role`, `is_premium`, `premium_expired_at`.
- **Billing checkout session**: `checkout_url`, `subscription_state`, `transaction_status`, dan amount breakdown (`original_amount`, `discount_amount`, `final_amount`); `coupon_code` bersifat opsional.
- **Billing status**: `subscription_state` (source of truth entitlement), `last_transaction_status`.
- **Jobs list**: `data[]` + `meta.pagination` (`page`, `limit`, `total_pages`, `total_records`).
- **Jobs detail**: `description` (plain text atau HTML dari source, wajib disanitasi saat render) dan `salary_range` (mendukung fallback exact/min-only/max-only/range; comparator ditampilkan sebagai label friendly `Up to`/`From`).
- **Preferences read/write**: `keywords`, `locations`, `job_types`, `salary_min`, `updated_at`.

## Output Implementasi Minimum per Modul

- Auth/session: auth pages, guard mechanism, single-flight refresh interceptor, integration tests `401` race.
- Discovery: jobs list/detail routes, URL query sync utility, tests untuk empty/error/rate-limit states.
- Preferences: form + normalizer + draft recovery behavior, tests validasi + reauth recovery.
- Billing/upgrade: checkout initiation + callback verify screen, tests untuk `pending_payment`/`premium_active`/gagal.
- Account/profile: profile summary + subscription badge precedence logic, test partial failure billing.

## Risiko Integrasi yang Perlu Dipantau

- Pastikan `GET /api/v1/preferences` mengembalikan default stabil untuk user baru agar bootstrap form tidak blocked.
- Field `auth/me.is_premium` dapat sementara berbeda dengan `billing/status.subscription_state`; entitlement UI harus mengikuti `billing/status`.
